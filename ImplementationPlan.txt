PHASE 2: DYNAMISCHER LEHRPLAN (DYNAMIC CURRICULUM)
--------------------------------------------------
Ziel: Der Trainer soll nicht mehr nur ein Thema ("Focus Area") haben, sondern eine Liste von Themen abarbeiten.

SCHRITTE:

1.  **Konfiguration erweitern (`training_config.json`)**
    *   Statt einem simplen String für `focus_area`, wird eine Liste von Aufgaben unterstützt.
    *   Struktur-Beispiel:
        ```json
        {
          "persona": "Kritischer User",
          "curriculum": [
            {"topic": "Philosophie & Moral", "duration_minutes": 120},
            {"topic": "Umgang mit wütenden Usern", "duration_minutes": 60},
            {"topic": "Allgemeinwissen & Smalltalk", "duration_minutes": "infinite"}
          ]
        }
        ```

2.  **Trainer Agent Update (`trainer_agent.py`)**
    *   Klasse `TrainerConfig` anpassen: `focus_area` wird zu `curriculum` (List).
    *   Neue Logik im `TrainerAgent`:
        *   `current_topic_index`: Trackt, wo wir im Lehrplan sind.
        *   `topic_start_time`: Trackt, wann das aktuelle Thema begann.
        *   Methode `get_current_focus()`: Prüft die Zeit und wechselt bei Bedarf zum nächsten Thema.
    *   System-Prompt-Update: Der System-Prompt wird bei jedem `generate_reply` Aufruf dynamisch mit dem *aktuellen* Fokus (aus `get_current_focus()`) gebaut.

3.  **Training Loop Integration (`training_loop.py`)**
    *   Visualisierung: In der Konsole anzeigen, welches Thema gerade aktiv ist ("Thema 1/3: Philosophie").
    *   Der Loop selbst muss wenig geändert werden, da der Trainer das Thema intern wechselt.
    *   Optional: Ein Event/Log, wenn das Thema gewechselt wird.

PHASE 3: MODELL-AUSWAHL & CEREBRAS INTEGRATION
----------------------------------------------
Ziel: Auswahl spezifischer Modelle (Lokal & API) vor dem Training.

SCHRITTE:

1.  **Konfiguration erweitern (`config/config.py`)**
    *   Neuen Provider im Enum `LLMProvider` hinzufügen: `CEREBRAS`.
    *   Neue Settings hinzufügen:
        *   `CEREBRAS_API_KEY` (in `secrets.py` und `config.py`).
        *   `CEREBRAS_MODEL` (Default z.B. "llama3.1-70b").

2.  **Brain Engine erweitern**
    *   Erstellen von `brain/cerebras_brain.py`.
    *   Implementation analog zu `GroqBrain` (nutzt OpenAI SDK kompatiblen Client).
        *   Base URL: `https://api.cerebras.ai/v1`
        *   Models: `llama3.1-70b`, `zai-glm-4.7` (sofern verfügbar), `qwen-3-235b...` (sofern im API Katalog).
    *   Update `brain/__init__.py`: `get_brain()` muss `CEREBRAS` unterstützen.

3.  **Setup Script Update (`setup_training.py`)**
    *   Die Auswahl-Logik am Anfang überarbeiten.
    *   Neue Menüstruktur:
        1. [API] Groq (Bestehend)
        2. [API] Cerebras (Neu - High Speed)
        3. [Lokal] Ollama
    *   Bei Auswahl "2. Cerebras":
        *   Sub-Menü Modelle: `zai-glm-4.7`, `qwen-3-235b-a22b-instruct-2507`, `llama-3.3-70b`.
    *   Bei Auswahl "3. Lokal":
        *   Sub-Menü Modelle: `llama3:70b`, `llama3:8b`, Custom Input.

4.  **Secrets Management**
    *   Sicherstellen, dass der Cerebras API Key abgefragt oder aus `config/secrets.py` geladen wird.
